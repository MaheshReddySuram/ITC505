Project Addendum

Development Process

1. Planning: I began by mapping out the story flow with 8 distinct endings. Each decision point leads to different outcomes, creating a branching narrative structure.

2. HTML Structure: Created a clean layout with separate sections for the story text, choices, and image display area.

3. JavaScript Logic: Implemented a story object containing all possible scenes, with properties for text, choices, and corresponding images.

4. CSS Styling: Used a vibrant color scheme with gradients and modern UI elements to make the game visually appealing.

Creative Choices

- Color Scheme: Selected a vibrant purple-to-teal gradient background with yellow accents to create an adventurous feel.

- Interactive Elements: Designed pill-shaped buttons with hover effects to make choices feel tangible.

- Visual Feedback: Added subtle animations and transitions to enhance user experience.

Challenges Faced

1. Story Complexity: Managing multiple branching paths required careful planning to ensure all endings were reachable.

2. Image Sizing: Ensuring consistent image display across different scenes required CSS object-fit properties.

3. Mobile Responsiveness: Adapting the two-column layout for mobile devices required media queries and reordering of elements.

Solutions Implemented

- Created a detailed story map before coding to visualize all possible paths.

- Used CSS flexbox and object-fit to handle varying image dimensions.

- Implemented responsive design with media queries to ensure good mobile experience.

Key Learnings

1. Planning complex branching narratives requires careful documentation.

2. CSS Grid and Flexbox are powerful tools for creating responsive layouts.

3. Visual feedback (like button hover states) significantly improves user experience.
